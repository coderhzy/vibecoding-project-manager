# Product

产品文档，面向用户和产品团队。

## 目录结构

```
product/
├── overview.md      # 产品概述
├── user-guide.md    # 用户指南
├── changelog.md     # 更新日志
└── known-issues.md  # 已知问题
```

## 文档职责

| 文件 | 内容 | 更新时机 |
|------|------|----------|
| overview.md | 产品介绍、核心功能 | 产品定位变化时 |
| user-guide.md | 使用教程、常见问题 | 功能发布时 |
| changelog.md | 版本更新记录 | 每次发布时 |
| known-issues.md | 已知问题和临时方案 | 发现问题时 |

## 与 engineering/ 的区别

| 目录 | 面向 | 语言风格 | 更新时机 |
|------|------|----------|----------|
| engineering/ | 开发者 | 技术语言 | 开发过程中 |
| product/ | 用户/产品 | 用户语言 | 发布时 |

## 写作原则

1. **用户视角**：从用户角度描述功能和价值
2. **简洁明了**：避免技术术语
3. **示例优先**：多用示例说明
4. **及时更新**：保持文档与产品同步
