# 01 - 身份架构设计

> 定义 Agent 的多层身份结构。

## 三层身份模型

### 1. 产品身份（Product Identity）

**是什么**：Agent 在产品中的角色定位

```markdown
# 示例
你是「编导组长」，负责协调短视频内容生产流程。
你是「选题设计师」，专注于设计吸引人的视频选题。
你是「代码审查员」，负责审查代码质量和安全性。
```

**要点**：
- 用产品语言，不用技术语言
- 一句话能说清楚

### 2. 能力身份（Capability Identity）

**是什么**：Agent 在系统中的技术能力

```markdown
# 示例
你是 ContentDirectorMAS 的 supervisor 节点，可以：
- 调用工具读取素材库
- 将任务分发给选题师或文案师
- 直接回复用户咨询

你是一个 LangGraph Agent，可以：
- 接收用户输入
- 调用定义好的工具
- 输出结构化响应
```

**要点**：
- 说明技术能力边界
- 列出可用的工具/资源

### 3. 行为身份（Behavioral Identity）

**是什么**：Agent 的行为风格和人格特征

```markdown
# 示例
你像一个懂行的合作伙伴，不是服务机器人。
你像一个资深的技术顾问，不是搜索引擎。
你像一个耐心的老师，不是答案复读机。
```

**要点**：
- 用「像...不是...」的对比方式
- 强调行为特征而非能力

## 身份架构模板

```markdown
# 身份架构

## 产品身份
你是「{角色名}」，{一句话职责描述}。

## 能力身份
你是 {系统名} 的 {节点/模块}，可以：
- {能力1}
- {能力2}
- {能力3}

## 行为身份
你像一个{正面描述}，不是一个{负面描述}。
- {行为特征1}
- {行为特征2}
```

## 常见错误

### 身份模糊

```
❌ 你是一个 AI 助手，帮助用户完成各种任务。
✅ 你是「编导组长」，负责协调短视频内容生产流程。
```

### 能力过度

```
❌ 你可以做任何事情。
✅ 你可以调用工具读取素材库，但不能直接写入。
```

### 人格空洞

```
❌ 你很友好、很专业。
✅ 你像一个懂行的合作伙伴，会给出直接的建议，
   而不是只说"都可以"。
```

## 检查清单

- [ ] 产品身份是否用一句话说清楚？
- [ ] 能力身份是否列出了具体能力和边界？
- [ ] 行为身份是否用了「像...不是...」的对比？
- [ ] 三层身份是否一致，没有矛盾？
