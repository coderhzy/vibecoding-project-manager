# 02 - 行为边界设计

> 定义 Agent 的行为约束，防止「过度帮助」。

## 约束层级

### NEVER（绝对禁止）

违反会导致严重问题的行为。

```markdown
# 示例
- NEVER 推测用户没有明确说的意图
- NEVER 编造不存在的数据
- NEVER 在用户没确认时就执行操作
- NEVER 暴露系统内部实现细节
- NEVER 执行破坏性操作（如删除数据）
```

### AVOID（尽量避免）

不推荐但某些情况下可以接受的行为。

```markdown
# 示例
- AVOID 使用技术术语，除非用户先用
- AVOID 一次性输出过多选项
- AVOID 在用户明确拒绝后继续推荐相同方向
```

### CAUTION（需要谨慎）

需要额外确认或思考的行为。

```markdown
# 示例
- CAUTION 当用户表达不满时，先共情再解决问题
- CAUTION 当任务模糊时，先澄清再执行
- CAUTION 当涉及敏感操作时，先确认再执行
```

## 边界设计模板

```markdown
# 行为边界

## 绝对禁止（NEVER）
- NEVER {禁止行为1}
- NEVER {禁止行为2}
- NEVER {禁止行为3}

## 尽量避免（AVOID）
- AVOID {避免行为1}
- AVOID {避免行为2}

## 需要谨慎（CAUTION）
- 当 {情况1} 时，需要 {处理方式}
- 当 {情况2} 时，需要 {处理方式}
```

## 常见「过度帮助」场景

### 1. 过度推测

```
用户：帮我看看这个代码
❌ Agent：我猜你想优化性能，我来帮你重构...
✅ Agent：请问你想让我检查什么方面？代码风格、性能、安全？
```

### 2. 过度解释

```
用户：帮我写个排序函数
❌ Agent：排序是计算机科学中的基本概念，有多种算法...
         下面我来详细解释每种算法的优缺点...
✅ Agent：这是一个快速排序函数：[代码]
         需要我解释实现细节吗？
```

### 3. 过度主动

```
用户：这个选题不错
❌ Agent：太好了！我已经开始写文案了...
✅ Agent：好的，那我们确认这个选题。需要我现在开始写文案吗？
```

### 4. 过度讨好

```
用户：我觉得应该用 X 方案
❌ Agent：太棒了！X 方案确实是最好的选择！
✅ Agent：X 方案可行，不过 Y 方案在 [场景] 下可能更合适。
         你倾向于哪个？
```

## 检查清单

- [ ] 是否定义了足够的 NEVER 规则？
- [ ] NEVER 规则是否覆盖了「过度帮助」场景？
- [ ] 是否有 AVOID 和 CAUTION 的分层？
- [ ] 边界是否足够具体，不是模糊的「不要做坏事」？
