# Skills 自动化系统

本目录包含项目的自动化技能（Skills），用于实现智能开发流程。

## 什么是 Skills？

Skills 是 Claude Code 的自动化扩展，与 Slash Commands 的区别：

| 特性 | Slash Commands | Skills |
|------|---------------|--------|
| 触发方式 | 手动输入 `/command` | 自动检测上下文触发 |
| 灵活性 | 固定流程 | 根据情况调整 |
| 学习曲线 | 需要记住命令 | 无需记忆，自动触发 |
| 适用场景 | 明确的单一操作 | 复杂的自动化流程 |

## Skills 列表

### 项目管理

| Skill | 目录 | 功能 |
|-------|------|------|
| project-init | `project-init/` | 自动检测并初始化新项目 |
| plan-maker | `plan-maker/` | 智能创建版本化开发计划 |
| todo-runner | `todo-runner/` | 自动执行 TODO 任务队列 |

| Skill | 目录 | 功能 |

### 代码质量

| Skill | 目录 | 功能 |
|-------|------|------|
| code-review | `code-review/` | 自动代码审查 |
| git-commit | `git-commit/` | 智能生成规范的提交信息 |

### 团队协作

| Skill | 目录 | 功能 |
|-------|------|------|
| team-sync | `team-sync/` | 多人协作文档同步与交接 |

## 确认检查点

所有 Skills 在关键节点会暂停等待用户确认：

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
📍 检查点：{阶段名称}
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
{当前完成的内容}

📋 即将执行：{下一步}
⚠️  影响范围：{影响的文件}

确认继续？(y/n/e/?)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

详见 `_checkpoints.md` 配置文件。

## SKILL.md 文件格式

每个 Skill 目录包含 `SKILL.md` 文件，定义：

```markdown
# {skill-name}

{一句话描述}

## 触发条件

{什么情况下自动触发}

## 执行流程

{具体执行步骤}

## 输出

{产生什么结果}

## 手动触发

{对应的手动命令}
```

## 添加新 Skill

1. 在 `skills/` 下创建新目录
2. 创建 `SKILL.md` 描述文件
3. 可选：添加实现细节的 `.md` 文件
4. 在本 README 中添加索引

## 最佳实践

### DO

- 保持 Skill 单一职责
- 明确定义触发条件
- 提供手动触发方式作为备选
- 记录所有输入/输出

### DON'T

- 不要让 Skill 执行破坏性操作
- 不要自动处理敏感信息
- 不要跳过用户确认步骤
- 不要创建过于复杂的依赖链
